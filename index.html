// 01 — AboutSobre

The professional who knows
where the data breaks. O profissional que sabe
onde o dado quebra.

I'm a Data & Analytics Engineer with over 5 years working inside the most operationally complex domains in business: Order-to-Cash, Procure-to-Pay, Manufacturing, Inventory, and International Trade. Not just analyzing data from them — living inside them.

That depth is my edge. I understand how a Purchase Order behaves differently across three ERP systems, why an invoice's status field means different things before and after month-end close, and where the data silently breaks before it ever reaches a dashboard.

I've led ERP implementations and transitions for NetSuite, Dynamics AX, Zoho, and OSGT — working as both Project Manager and Key User. That experience means I design data models with source systems in mind, not against them.

My deliverables get used. That's the metric that matters.

// 03 — Stack

Tools I use to build
things people trust. Ferramentas para construir
coisas em que confiam.

dbt

I use dbt to enforce business logic as code: modular models, documented contracts, tested metrics. If a KPI goes to production without a test, it's not ready for production. Uso dbt para transformar lógica de negócio em código: modelos modulares, contratos documentados, métricas testadas. Se um KPI vai para produção sem teste, não está pronto para produção.

BigQuery / GCP

My primary analytical layer. I design schemas with query cost and performance in mind — partitioning, clustering, and incremental models are defaults, not afterthoughts. Minha camada analítica principal. Projeto schemas com custo e performance em mente — particionamento, clustering e modelos incrementais são padrão, não exceção.

Airflow

I build pipelines that are observable by design: explicit dependencies, alerting, documentation baked in. Not pipelines that run until they mysteriously don't. Construo pipelines observáveis por design: dependências explícitas, alertas e documentação integrados. Não pipelines que funcionam até misteriosamente pararem.

Looker / Power BI

I build for adoption. A dashboard no one uses is a failed project. Every report I deliver has a defined owner, a defined question, and a defined action it enables. Construo para adoção. Um dashboard que ninguém usa é um projeto fracassado. Todo relatório que entrego tem um dono, uma pergunta e uma ação esperada.

SQL & Python

SQL is how I think about data. Python is how I automate the parts that shouldn't be manual. I use both where they're strongest, not where they're fashionable. SQL é como penso sobre dados. Python é como automatizo o que não deveria ser manual. Uso ambos onde são mais fortes, não onde estão na moda.

ERP Systems

NetSuite, Dynamics AX, Zoho, OSGT — I've been inside these as Key User and PM. I know how source data is generated, not just how to query it. That changes everything about how I model it. NetSuite, Dynamics AX, Zoho, OSGT — estive dentro como Key User e PM. Sei como o dado de origem é gerado, não só como consultá-lo. Isso muda tudo na hora de modelar.
